# IBM i RPG III プログラミング規約

## プロジェクト概要
このプロジェクトは、IBM i (AS/400) 上で動作するRPG IIIプログラムの開発・保守を行います。
既存のレガシーコードベースを参考に、一貫性のあるコーディングスタイルを維持してください。

## 参考プログラム
- `QEOL400/QEOLRPG/bch110.rpg`: 得意先マスター一覧表（基本的なバッチプログラムの雛形）
- `QEOL400/QEOLRPG/bch130.rpg`: 得意先別売上高一覧表（計算処理を含むバッチプログラム）

## コーディング規約

### 1. ファイル構造
RPG IIIプログラムは以下の順序で記述してください：
1. メタデータ（`*%METADATA ... *%EMETADATA`）
2. H仕様書（ヘッダー仕様）
3. F仕様書（ファイル定義）
4. C仕様書（計算処理）
5. O仕様書（出力定義）

### 2. メタデータ
```rpg
      *%METADATA                                                       *
      * %TEXT プログラムの説明文
      *%EMETADATA                                                      *
```

### 3. ヘッダー仕様（H仕様書）
```rpg
     H            Y
```
- 常に `Y` を指定（日付編集コード）

### 4. ファイル定義（F仕様書）
```rpg
     F*コメント：ファイルの説明
     FファイルIF  E           K        DISK
     FQPRINT  O   F     132     OF     PRINTER
```
- コメント行は `F*` で開始
- 入力ファイル：`IF E K DISK`
- 印刷ファイル：`O F 132 OF PRINTER`

### 5. 計算処理（C仕様書）

#### 5.1 コメント
```rpg
     C*処理の説明
```
- 各処理ブロックの前に日本語コメントを記述

#### 5.2 変数の初期化
```rpg
     C                     Z-ADD0        変数名  桁数
```
- 数値変数の初期化には `Z-ADD0` を使用
- 桁数指定は必須（例：90 = 9桁整数、72 = 7桁小数点以下2桁）

#### 5.3 ループ処理
```rpg
     C           *IN99     DOWEQ'0'
     C                     READ ファイル名                 99
     C           *IN99     IFEQ '0'
     C*処理内容
     C                     END
     C                     END
```
- ファイル終了インジケーターは `*IN99` を使用
- `DOWEQ'0'` でループ開始
- `READ` 操作でインジケーター99をセット

#### 5.4 条件分岐
```rpg
     C           フィールド IFGT 0
     C*真の場合の処理
     C                     ELSE
     C*偽の場合の処理
     C                     END
```

#### 5.5 算術演算
```rpg
     C           変数1     ADD  変数2    結果    桁数
     C           変数1     SUB  変数2    結果    桁数
     C           変数1     MULT 変数2    結果    桁数
     C           変数1     DIV  変数2    結果    桁数
```
- **重要**: 結果フィールドの桁数指定は必須
- 小数点を含む計算では適切な桁数を指定（例：72 = 小数点以下2桁）
- 整数のみの場合は90などを指定

#### 5.6 ゼロ除算エラー処理
```rpg
     C           除数      IFGT 0
     C           被除数    DIV  除数     結果    72
     C                     ELSE
     C                     Z-ADD0        結果    72
     C                     END
```
- 除算の前に必ず除数がゼロでないことを確認

#### 5.7 頁あふれ処理
```rpg
     C   OF                EXCPT見出し名
     C   OF                SETOF                     OF
```

#### 5.8 出力処理
```rpg
     C                     EXCPT出力名
```

#### 5.9 終了処理
```rpg
     C                     SETON                     LR
     C                     RETRN
```

### 6. 出力定義（O仕様書）

#### 6.1 見出し行
```rpg
     OQPRINT  E  202           見出し名
     O                                   位置 '文字列'
     O                         フィールド Y   位置
     O                         PAGE  Z  位置
```
- `E 202`: 改ページ後2行スキップ
- `Y`: 日付編集コード
- `Z`: ゼロサプレス

#### 6.2 明細行
```rpg
     O        E  1             明細名
     O                         フィールド    位置
```
- `E 1`: 1行スキップ

#### 6.3 編集コード
```rpg
     O                         フィールドJ   位置
```
- `J`: カンマ編集（数値をカンマ区切りで表示）

### 7. 数値フィールドの桁数指定ルール

**重要**: RPG IIIでは数値フィールドの桁数指定が結果の精度を決定します。

#### 桁数指定の形式
- `90`: 9桁の整数（小数点なし）
- `72`: 7桁、小数点以下2桁（例：12345.67）
- `92`: 9桁、小数点以下2桁（例：1234567.89）

#### 計算結果の桁数指定
```rpg
C           1200000   DIV  1000000   WKZNHI  72
```
- 上記の例では、結果は `1.20` となります
- もし `90` を指定すると、結果は `1` に切り捨てられます

#### パーセント計算の例
```rpg
C           TKZURI    IFGT 0
C           TKNURI    DIV  TKZURI    WKZNHI  72
C           WKZNHI    MULT 100       WKZNHI  72
C                     ELSE
C                     Z-ADD0         WKZNHI  72
C                     END
```
- 前年比 = (当年 ÷ 前年) × 100
- 小数点以下2桁の精度を維持するため `72` を指定

### 8. 命名規則

#### ファイル名
- バッチプログラム: `bch###.rpg` (例: bch110.rpg, bch130.rpg)
- 対話プログラム: `iph###.rpg`

#### フィールド名
- マスターファイルのフィールド: `TK` で開始（例: TKBANG, TKNAKJ）
- ワーク変数: `WK` で開始（例: WKGOTN, WKZNHI）

#### インジケーター
- ファイル終了: `*IN99`
- 頁あふれ: `OF`
- 最終レコード: `LR`

### 9. コメント規約

#### 日本語コメント
- すべてのコメントは日本語で記述
- 各処理ブロックの前に処理内容を説明

#### コメントの例
```rpg
C*ワーク項目の初期化
C*見出し印刷
C*得意先マスターの読み込み（売上高順）
C*前年比の計算（ゼロ除算エラー処理）
C*合計の累計
C*頁変えの確認
C*明細の印刷
C*合計行の印刷
C*終了処理
```

### 10. 印刷レイアウト

#### タイトル行
```rpg
O                                   31 '＊＊＊　タイトル'
O                                   53 'サブタイトル'
O                                   69 '　　＊＊＊'
O                                   87 '作成日：'
O                         UDATE Y   98
O                                  113 '頁'
O                         PAGE  Z  118
```

#### 項目見出し
```rpg
O        E  2             MIDASI
O                                   位置 '項目名1'
O                                   位置 '項目名2'
```

#### 明細行
```rpg
O        E  1             MEISAI
O                         フィールド    位置
```

### 11. エラー処理

#### 必須のエラー処理
1. **ゼロ除算**: 除算前に除数がゼロでないことを確認
2. **ファイル終了**: `*IN99` で適切に処理
3. **頁あふれ**: `OF` インジケーターで見出しを再印刷

### 12. プログラム生成時の注意事項

#### LAB3の要件に基づくプログラム生成
1. **参考プログラムの確認**: bch110.rpg のスタイルを踏襲
2. **完全なプログラム生成**: F, C, O 仕様書をすべて含める
3. **コメントの充実**: 各処理ブロックに日本語コメントを追加
4. **数値精度の確保**: 計算結果の桁数指定を適切に行う
5. **エラー処理の実装**: ゼロ除算などのエラーケースを考慮

#### 特に重要なポイント
- **小数点を含む計算では必ず適切な桁数（例：72）を指定**
- パーセント計算などで整数（90）を指定すると、小数点以下が切り捨てられる
- 出力仕様書の編集コード（J）は、フィールドの精度に従って自動的にフォーマット

### 13. 印刷フォーマットの調整

#### 文字列の配置
- 全角スペースを使用して文字間隔を調整可能
- 例: `'＊＊＊　得　意　先　別'` → `'＊＊＊得意先別'`（スペース削除で詰める）

#### 列見出しの位置
- 数値項目は右寄せ
- 文字項目は左寄せ
- 位置指定で微調整可能

## まとめ

このルールに従うことで、既存のコードベースと一貫性のある、保守性の高いRPG IIIプログラムを作成できます。特に数値計算の桁数指定は、プログラムの正確性に直結する重要な要素です。必ず適切な桁数を指定してください。

## 14. よくあるコンパイルエラーと対処法

このセクションは、BCH130プログラム作成時に実際に遭遇したエラーと解決方法をまとめたものです。

### 14.1 E仕様書（拡張仕様書）の誤用

**エラー**:
```
QRG5014 演算命令の指定が正しくない
QRG7030 フィールドまたは標識が定義されていない
```

**原因**:
```rpg
E*ワーク項目の定義
E                    WKGOTN         90
E                    WKGOZE         90
E                    WKZNHI         72
```
固定形式RPG IIIでは、E仕様書でワーク項目を定義できません。E仕様書は配列やテーブルの定義に使用されます。

**正しい方法**:
```rpg
C*ワーク項目の初期化
C                     Z-ADD0         WKGOTN  90
C                     Z-ADD0         WKGOZE  90
```
- ワーク項目はC仕様書でZ-ADD命令を使って初期化する
- 初期化と同時に桁数を定義する

### 14.2 算術演算の結果フィールド桁数指定漏れ

**エラー**:
```
QRG5024 演算項目2の指定が正しくありません
QRG5031 結果のフィールドの指定が正しくない
```

**誤った例**:
```rpg
C           TKNURI    DIV  TKZURI   WKZNHI
C           WKZNHI    MULT 100      WKZNHI
C                     Z-ADD0        WKZNHI
```

**正しい例**:
```rpg
C           TKNURI    DIV  TKZURI    WKZNHI  72
C           WKZNHI    MULT 100       WKZNHI  72
C                     Z-ADD0         WKZNHI  72
```
- **すべての算術演算で結果フィールドの桁数指定が必須**
- DIV、MULT、ADD、SUB、Z-ADDすべてに適用
- 桁数指定を忘れるとコンパイルエラーになる

### 14.3 桁数指定の位置とスペース

**重要**: 桁数指定は結果フィールド名の直後に記述し、適切なスペースを入れる

**正しい書き方**:
```rpg
C                     Z-ADD0         WKGOTN  90
C           TKNURI    DIV  TKZURI    WKZNHI  72
```
- 結果フィールド名と桁数の間にスペースを入れる
- 桁数は2桁で記述（90、72など）

### 14.4 小数点精度の選択

**整数計算の場合**:
```rpg
C           WKGOTN    ADD  TKNURI    WKGOTN  90
```
- 小数点が不要な場合は `90`（9桁整数）

**小数点を含む計算の場合**:
```rpg
C           TKNURI    DIV  TKZURI    WKZNHI  72
C           WKZNHI    MULT 100       WKZNHI  72
```
- 小数点以下2桁が必要な場合は `72`（7桁、小数点以下2桁）
- パーセント計算など、精度が重要な場合は必ず小数点付きの桁数を指定

### 14.5 印刷フォーマットの文字列調整

**bch130回答.rpgの例**:
```rpg
O                                   31 '＊＊＊得意先別'
O                                   53 '売上高一覧表'
O                                   69 '＊＊＊'
```

**ポイント**:
- 全角スペースを削除して文字を詰めることができる
- `'＊＊＊　得　意　先　別'` → `'＊＊＊得意先別'`
- 印刷位置を調整して見やすいレイアウトにする

### 14.6 デバッグのヒント

**コンパイルエラーが出た場合のチェックリスト**:
1. [ ] すべての算術演算に桁数指定があるか
2. [ ] Z-ADD命令に桁数指定があるか
3. [ ] E仕様書でワーク項目を定義していないか
4. [ ] 結果フィールド名と桁数の間にスペースがあるか
5. [ ] 小数点を含む計算で適切な桁数（72など）を指定しているか

**参考プログラムとの比較**:
- bch110.rpg: 基本的な構造の参考
- bch120.rpg: Z-ADD命令の使い方の参考
- bch130回答.rpg: 正しい実装例

## 15. プログラム作成のベストプラクティス

### 15.1 段階的な開発アプローチ

1. **参考プログラムの確認**: 既存のbch110.rpgなどを確認
2. **ファイル構造の作成**: H、F仕様書から開始
3. **基本ロジックの実装**: C仕様書の主要処理
4. **エラー処理の追加**: ゼロ除算などの例外処理
5. **出力定義の作成**: O仕様書の印刷レイアウト
6. **コンパイルとテスト**: エラーを修正しながら完成

### 15.2 コメントの重要性

**良いコメントの例**:
```rpg
C*ワーク項目の初期化
C                     Z-ADD0         WKGOTN  90
C                     Z-ADD0         WKGOZE  90
C*見出し印刷
C                     EXCPTMIDASI
C*得意先マスターの読み込み（売上高順）
C                     READ TOKMSL01                 99
C*前年比の計算（ゼロ除算エラー処理）
C           TKZURI    IFGT 0
```

**ポイント**:
- 各処理ブロックの前に必ずコメントを記述
- 処理の目的を明確に説明
- 特殊な処理（エラー処理など）は詳しく説明

### 15.3 保守性を高める工夫

1. **一貫した命名規則**: WKで始まるワーク変数、TKで始まるマスターフィールド
2. **適切なインデント**: 条件分岐やループの構造を視覚的に明確に
3. **エラー処理の徹底**: ゼロ除算、ファイル終了、頁あふれなど
4. **コメントの充実**: 将来の保守担当者が理解しやすいように

## 16. まとめ：このタスクで学んだ重要なポイント

### 最も重要な教訓

1. **E仕様書の誤用を避ける**
   - 固定形式RPG IIIでは、ワーク項目をE仕様書で定義できない
   - C仕様書でZ-ADD命令を使って初期化する

2. **桁数指定は必須**
   - すべての算術演算（DIV、MULT、ADD、SUB、Z-ADD）で桁数指定が必要
   - 小数点を含む計算では適切な桁数（72など）を指定

3. **参考プログラムの活用**
   - 既存のプログラム（bch110.rpg、bch120.rpg）を参考にする
   - 同じスタイルとパターンを踏襲する

4. **段階的な開発とテスト**
   - 一度にすべてを作成せず、段階的に実装
   - コンパイルエラーを一つずつ解決

これらのルールとベストプラクティスに従うことで、エラーの少ない、保守性の高いRPG IIIプログラムを効率的に作成できます。
