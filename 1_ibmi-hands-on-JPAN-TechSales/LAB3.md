# LAB3: 新規プログラムの作成 ✨

## 🎯 このラボの目標

IBM Bobを使って、既存プログラムを参考に新規バッチプログラムを作成する方法を学びます。ゼロからの開発を効率化するスキルを習得します。

**所要時間**: 10-15分  
**難易度**: ★★★☆☆（中級）  
**使用モード**: Orchestrator モード / Code モード

---

## 📚 学習内容

このラボでは、以下のスキルを習得します：

- ✅ プログラム仕様書の自動生成
- ✅ コードの段階的な生成
- ✅ 既存プログラムのパターン踏襲
- ✅ 自動コードレビュー
- ✅ 効率的な開発ワークフロー

## 🚀 ステップ1: 事前準備（2分）

### 1.1 参考ファイルを開く

以下のファイルを開きます：

```
1_ibmi-hands-on-JPAN-TechSales/QEOL400/QEOLRPG/bch110.rpg      # 参考にする既存プログラム
1_ibmi-hands-on-JPAN-TechSales/QEOL400/QEOLDBF/tokmsp.pf       # 得意先マスター定義
```

### 1.2 新規プログラムの要件確認

**作成するプログラム**: BCH120（得意先別売上高一覧表）

**要件**:
- 得意先マスターから当年売上高（TKNURI）を読み込む
- 売上高の高い順に並べ替えて印刷
- 合計金額を最後に印刷

---

## 💡 ステップ2: プログラム仕様の作成（3分）

### 2.1 仕様書の生成

IBM Bobに要件を伝えて、プログラム仕様書を作成してもらいましょう。

**Bobへの質問例**:
```
BCH110を参考に、以下の要件で新規プログラムBCH120を作成したいです。

【要件】
- プログラム名: BCH120
- 機能: 得意先別売上高一覧表の印刷
- 入力: 得意先マスター（TOKMSL01 - 売上高順）
- 出力: 印刷ファイル（QPRINT）
- 印刷項目:
  * 得意先番号（TKBANG）
  * 得意先名（TKNAKJ）
  * 当年売上高（TKNURI）
  * 前年売上高（TKZURI）
  * 前年比（計算項目）
- 処理:
  1. 見出し印刷
  2. 得意先マスターを売上高順に読み込み
  3. 前年比を計算（当年÷前年×100）
  4. 明細印刷
  5. 合計行印刷

まず、プログラム仕様書を作成してください。
```

**期待される回答のポイント**:
- プログラム概要
- ファイル定義
- 処理フロー
- データ項目定義

💡 **ヒント**: 仕様書をドキュメントとして保存しておきましょう。

### 2.2 処理フローの確認

**Bobへの質問例**:
```
BCH120の処理フローをフローチャートで表現してください。
BCH110との違いを明確にしてください。
```

## 🔍 ステップ3: ファイル定義の作成（2分）

### 3.1 F仕様書の生成

**Bobへの質問例**:
```
BCH120のF仕様書（ファイル定義）を作成してください。
以下の点に注意してください：
- 入力ファイル: TOKMSL01（売上高順の論理ファイル）
- 出力ファイル: QPRINT（132桁の印刷ファイル）
- BCH110の形式に合わせる
```

**期待される回答例**:
```rpg
     H            Y
     F*BCH120得意先別売上高一覧表の印刷
     FTOKMSL01IF  E           K        DISK
     FQPRINT  O   F     132     OF     PRINTER
```

### 3.2 作業項目の定義

**Bobへの質問例**:
```
BCH120で使用する作業項目を定義してください：
- 前年比（WKZENB）: 3桁、小数点1桁
- 当年売上合計（WKTGOK）: 9桁
- 前年売上合計（WKZGOK）: 9桁
```

## 🛠️ ステップ4: 計算仕様の作成（3分）

### 4.1 メインループの生成

**Bobへの質問例**:
```
BCH120のメインループ処理（C仕様書）を作成してください。
BCH110を参考に、以下の処理を含めてください：
1. 見出し印刷
2. ファイル読み込みループ
3. 前年比の計算
4. 合計の累計
5. 明細印刷
6. 頁あふれ処理
```

**期待される回答例**:
```rpg
     C*見出し印刷
     C                     EXCPTMIDASI
     C*合計の初期化
     C                     Z-ADD0         WKTGOK
     C                     Z-ADD0         WKZGOK
     C           *IN99     DOWEQ'0'
     C*得意先マスターの読み込み
     C                     READ TOKMSL01                 99
     C           *IN99     IFEQ '0'
     C*前年比の計算
     C           TKZURI    IFGT 0
     C           TKNURI    DIV  TKZURI    WKZENB
     C           WKZENB    MULT 100       WKZENB
     C                     ELSE
     C                     Z-ADD0         WKZENB
     C                     END
     C*合計の累計
     C                     ADD  TKNURI    WKTGOK
     C                     ADD  TKZURI    WKZGOK
     C*頁変えの確認
     C   OF                EXCPTMIDASI
     C   OF                SETOF                     OF
     C*明細の印刷
     C                     EXCPTMEISAI
     C                     END
     C                     END
     C*合計行の印刷
     C                     EXCPTGOUKEI
     C*終了処理
     C                     SETON                     LR
     C                     RETRN
```

### 4.2 計算ロジックの説明

**Bobへの質問例**:
```
前年比の計算ロジックについて説明してください。
ゼロ除算のエラー処理はどのように実装されていますか？
```

💡 **ヒント**: エラー処理の実装方法を理解しておきましょう。

## 💡 このラボで学んだこと

### IBM Bobでできたこと

✅ **ゼロからのプログラム作成が劇的に早い**
- 従来: 仕様書作成→設計→コーディング（数時間～数日）
- Bob活用: 要件を伝えて、数分でプロトタイプ完成

✅ **既存コードのパターンを踏襲**
- BCH110の構造を理解して、同じスタイルで生成
- コーディング規約の統一が容易

✅ **レビューとブラッシュアップが簡単**
- 生成されたコードをその場でレビュー
- 改善点を指摘して、即座に修正

### 実務での活用シーン

1. **プロトタイプ開発**
   - 要件定義段階で、動くプロトタイプを素早く作成
   - 顧客との認識合わせに活用

2. **類似プログラムの量産**
   - 1つのプログラムを作れば、類似プログラムは数分で作成
   - 保守性の高いコードを量産

3. **レガシーコードのモダナイゼーション**
   - RPG IIIをRPG IVに変換
   - 古い処理を新しい方式に書き換え

## 🎯 よくある質問

**Q1: Bobが生成したコードはそのまま使える？**

A: 基本的には使えますが、必ずレビューが必要です。業務ロジックの妥当性は人間が確認してください。

**Q2: 複雑な処理も生成できる？**

A: 要件を明確に伝えれば可能です。段階的に機能を追加していく方が確実です。

**Q3: 既存のコーディング規約に従わせるには？**

A: 参考にするプログラムを明示してください。「BCH110のスタイルに合わせて」と指示することで、既存のパターンを踏襲したコードを生成できます。

## 📝 プログラム生成のテンプレート

### 基本的な依頼
```
[参考プログラム]を参考に、以下の要件で新規プログラムを作成してください：

【プログラム名】
[プログラムID]

【機能】
[機能説明]

【入力ファイル】
- [ファイル名]: [説明]

【出力ファイル】
- [ファイル名]: [説明]

【処理内容】
1. [処理1]
2. [処理2]
...

【印刷項目】
- [項目名]: [説明]
...
```

### 段階的な生成
```
ステップ1: F仕様書を作成してください
ステップ2: メインループを作成してください
ステップ3: 計算処理を追加してください
ステップ4: 出力仕様を作成してください
ステップ5: 完全なプログラムを統合してください
```

### レビュー依頼
```
以下のプログラムをレビューしてください：

[生成されたコード]

確認ポイント：
1. 構文エラー
2. ロジックエラー
3. パフォーマンス
4. 保守性
5. セキュリティ
```
