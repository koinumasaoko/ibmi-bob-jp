## 🚀 ステップ1: 事前準備（2分）

### 1.1 参考ファイルの確認（オプション）

以下のファイルを**エクスプローラーエリア**で確認します（参考用）：

```

QEOL400/QEOLDBF/tokmsp.pf       # 得意先マスター定義
```

💡 **ヒント**: これらのファイルはBobが自動的に参照するため、今開く必要はありません。既存プログラムの構造を事前に確認したい場合のみ参照してください。

1. **チャットウィンドウの左下**から**Code モード**を選択します
2. このラボでは、コード生成とレビューを中心に進めます

### 1.3 新規プログラムの要件確認

**作成するプログラム**: BCH130（得意先別売上高一覧表）

**要件**:
- 得意先マスターから当年売上高（TKNURI）を読み込む
- 売上高の高い順に並べ替えて印刷
- 前年比を計算して表示
- 合計金額を最後に印刷

## 💡 ステップ2: プログラムの一括生成（5分）

### 2.1 完全なプログラムの生成

IBM Bobに要件を伝えて、完全なプログラムを一気に生成してもらいましょう。

**Bobへの質問例**:
```
QEOL400/QEOLRPG/bch110.rpgを参考に、以下の要件で新規プログラムBCH130を作成してください。

【要件】
- プログラム名: BCH130
- 機能: 得意先別売上高一覧表の印刷
- 入力: 得意先マスター（TOKMSL01 - 売上高順）
- 出力: 印刷ファイル（QPRINT）
- 印刷項目:
  * 得意先番号（TKBANG）
  * 得意先名（TKNAKJ）
  * 当年売上高（TKNURI）
  * 前年売上高（TKZURI）
  * 前年比（計算項目: 当年÷前年×100）
- 処理:
  1. 見出し印刷（タイトル、作成日、ページ番号、項目見出し）
  2. 得意先マスターを売上高順に読み込み
  3. 前年比を計算（ゼロ除算エラー処理を含む）
  4. 明細印刷
  5. 合計行印刷（当年売上合計、前年売上合計）
  6. 頁あふれ処理

QEOL400/QEOLRPG/bch110.rpgのスタイルに合わせて、完全なRPGプログラムを生成してください。
コメントも含めてください。
```

**期待される結果**:
- F仕様書（ファイル定義）
- D仕様書（データ定義）
- C仕様書（計算処理）
- O仕様書（出力定義）

すべてが含まれた完全なプログラムが生成されます。

💡 **ヒント**: 生成されたコードは、新しいファイル `QEOL400/QEOLRPG/bch130.rpg` として保存しましょう。

### 2.2 生成されたコードの確認

生成されたコードの主要部分を確認します：

**確認ポイント**:
- [ ] ファイル定義が正しいか（TOKMSL01、QPRINT）
- [ ] 前年比の計算ロジックが含まれているか
- [ ] ゼロ除算エラー処理があるか
- [ ] 見出し、明細、合計の出力定義があるか
- [ ] 頁あふれ処理が実装されているか

## 🎨 ステップ4: 印刷レイアウトの改善（3分）

### 4.1 PRTFファイルの作成

O仕様書で定義していた印刷レイアウトを、より保守性の高いPRTF（印刷ファイル）に変換します。

**Bobへの質問例**:
```
BCH130のO仕様書の印刷レイアウトを、PRTFファイル（BPL130P.PRTF）に変換してください。

【要件】
- ファイル名: BPL130P.PRTF
- レコード形式:
  * MIDASI: 見出し行（タイトル、作成日、ページ番号、項目見出し）
  * MEISAI: 明細行（得意先情報、売上高、前年比）
  * GOUKEI: 合計行（当年合計、前年合計）
- 印刷位置とフィールド長はO仕様書と同じにする
- 編集コードJを使用して数値を見やすく表示

PRTFファイルのDDSソースを生成してください。
```

**期待される結果**:
```

A                                  69 '　　　　　＊＊＊'

A            UDATE         Y      98
A                                 113 '頁'
A            PAGE          Z     118
A          R MEISAI
A            TKBANG              10
A            TKNAKJ              35
A            TKNURI        J     60
A            TKZURI        J     80
A            WKZENB        J    100
A          R GOUKEI

```

### 4.2 プログラムの修正

PRTFファイルを使用するようにBCH130を修正します。

**Bobへの質問例**:
```
BCH130プログラムを、BPL130P.PRTFを使用するように修正してください。

【変更点】
- F仕様書: QPRINT → BPL130P
- O仕様書を削除
- EXCPT命令でレコード形式を指定（MIDASI、MEISAI、GOUKEI）

修正されたプログラムを提示してください。
```

💡 **ヒント**: PRTFを使うことで、印刷レイアウトの変更がプログラムの再コンパイルなしで可能になります。

## 🎯 よくある質問

**Q1: Bobが生成したコードはそのまま使える？**

A: 基本的には使えますが、必ずレビューが必要です。特に業務ロジックの妥当性は人間が確認してください。AIは構文やパターンは理解していますが、業務要件の細かいニュアンスまでは理解できない場合があります。

**Q2: 複雑な処理も生成できる？**

A: 要件を明確に伝えれば可能です。ただし、非常に複雑な処理は段階的に機能を追加していく方が確実です。まず基本機能を作り、動作確認してから機能を追加するのがベストプラクティスです。

**Q3: 既存のコーディング規約に従わせるには？**

A: 参考にするプログラムを明示してください。「QEOL400/QEOLRPG/bch110.rpgのスタイルに合わせて」と指示することで、既存のパターンを踏襲したコードを生成できます。

**Q4: PRTFとO仕様書、どちらを使うべき？**

A: 新規開発ではPRTFを推奨します。理由：
- 印刷レイアウトの変更がプログラムの再コンパイルなしで可能
- レイアウトとロジックの分離で保守性が向上
- 複数のプログラムで同じPRTFを共有可能

ただし、既存システムとの整合性を考慮して選択してください。

**Q5: 複数のファイルを同時に作成したい場合は？**

A: 複数のファイル（RPG、PRTF、DSPFなど）を同時に作成する場合は、Orchestratorモードの使用を検討してください。Orchestratorモードは複数のステップを自動化し、関連ファイルを一括で作成できます。
