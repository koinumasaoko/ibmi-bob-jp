# IBM Bob ワークショップへようこそ 🚀

IBM Bobを使って、IBM i開発の生産性を劇的に向上させる方法を学ぶワークショップです。プログラミング経験は不問、AI SDLCパートナー「IBM Bob」が、あなたのソフトウェア開発ライフサイクル全体をサポートします。

## 🎯 ワークショップ概要

このワークショップでは、IBM Bobの主要機能を実際に使いながら、IBM i開発プロセスの効率化を体験します。

### 学べること

- ✅ IBM Bobの基本的な使い方と画面操作
- ✅ 既存RPGコードの理解と解析（Code モード）
- ✅ プログラムへの機能追加と修正（Code モード）
- ✅ 新規プログラムの作成（Code モード）
- ✅ AIによるコードレビューと品質向上
- ✅ 開発作業を効率的に進めるワークフロー

### 対象者

- 🎯 **IBM i 開発者** - 日々の開発を効率化したい
- 🎯 **プロジェクトマネージャー** - 開発プロセスの理解を深めたい
- 🎯 **エンジニア** - レガシーコードの保守に関心のある方
- 🎯 **IBM Bob初心者** - 実際の使用例から学習したい

## 📥 このハンズオン資料の取得方法

このハンズオン資料とサンプルプログラムは、以下の方法で取得できます：

### 方法1: Bobを使ってGit cloneで取得（推奨）

**📋 前提条件**
- 上記「事前準備チェックリスト」の「Git clone を使用する場合」の項目を満たしていること

**手順A: Bobに依頼する（最も簡単）**

Bobのチャットで以下のように依頼してください：

```
このリポジトリをクローンしてください：
https://github.com/koinumasaoko/ibmi-bob-jp.git
```

Bobが自動的に以下を実行します：
- `git clone`コマンドの実行
- クローン完了の確認
- ディレクトリ構造の確認

**手順B: Bob IDEのコマンドパレットを使用**

1. Bob IDEを開く
2. `Ctrl + Shift + P`（Mac: `Cmd + Shift + P`）でコマンドパレットを開く
3. 「Git: Clone」と入力して選択
4. リポジトリURLを入力：
   ```
   https://github.com/koinumasaoko/ibmi-bob-jp.git
   ```
5. クローン先のフォルダを選択
6. クローン完了後、「Open」をクリックしてプロジェクトを開く

**💡 Bobを使うメリット**
- ✅ コマンド不要 - Git コマンドを覚える必要なし
- ✅ エラー対応 - 問題が発生した場合、Bobが自動で対処
- ✅ 確認作業 - クローン後のファイル構造を自動確認

> **注意**: GitHubアカウントをお持ちでない場合、または設定が難しい場合は、「方法2: ZIPファイルでダウンロード」をご利用ください。

### 方法2: ZIPファイルでダウンロード

1. [GitHubリポジトリページ](https://github.com/koinumasaoko/ibmi-bob-jp.git)にアクセス
2. 緑色の「Code」ボタンをクリック
3. 「Download ZIP」を選択
4. ダウンロードしたZIPファイルを解凍
5. `1_ibmi-hands-on-JPAN-TechSales`フォルダを開く

## 📚 ワークショップの構成

このワークショップは3つのセクションで構成されています：

### 1️⃣ はじめに

IBM Bobの概要を学び、開発環境を整えます。

- **事前準備とセットアップ** - ワークショップの概要と必須環境
- **IBM Bobの紹介** - 機能とメリットの理解
- **Bobのインターフェース** - 画面構成と基本操作

### 2️⃣ ハンズオン

実際にIBM Bobを使って、IBM i開発を体験します。

#### Lab 1: 既存プログラムの理解 📖
**Ask モード → Code モードを使用**

既存のRPGバッチプログラム（BCH110）をIBM Bobで理解します。プログラムの処理フロー、ファイル定義、業務ロジックを質問しながら理解し、フローチャートなどのドキュメントを生成する方法を学びます。

📝 [Lab 1を始める](./LAB1.md)

#### Lab 2: プログラムの修正と影響分析 🔧
**Code モードを使用**

Lab 1で理解したプログラムに、新しい項目を追加します。対話型プログラム（IPH210）の画面ファイルとRPGプログラムを修正し、既存プロジェクトへの機能追加という、実務でよくあるシナリオを体験します。**影響範囲の分析（Impact Analysis）**の重要性も学びます。

📝 [Lab 2を始める](./LAB2.md)

#### Lab 3-FIX: 新規プログラムの作成（固定形式RPG） 🆕 (オプション・工事中 🚧)
**Code モードを使用**

既存プログラムのパターンを踏襲して、新規バッチプログラムを作成します。要件定義から実装、コードレビュー、印刷レイアウトの改善まで、IBM Bobを活用した実践的な開発フローを学びます。

> **🚧 工事中**: このラボは現在改善作業中です。内容が変更される可能性があります。

> **📌 注意**: LAB3-FIXはRPG III固定形式の新規開発（コピー新規）にフォーカスしています。LAB1（既存コードの理解）とLAB2（既存コードの修正）で基本を学んだ後、RPG III固定形式の新規開発（コピー新規）学習が必要な場合のみ実施してください。

📝 [Lab 3-FIXを始める](./LAB3-FIX.md)

#### Lab 3-FF: 固定形式から自由形式への変換と機能追加 🔄 (オプション・工事中 🚧)
**Code モードを使用**

RPG III固定形式のプログラムを自由形式RPG（RPG IV/ILE）に変換し、さらに新機能を追加する方法を学びます。

> **🚧 工事中**: このラボは現在作成中です。完成までお待ちください。

📝 [Lab 3-FFを始める](./LAB3-FF.md)
