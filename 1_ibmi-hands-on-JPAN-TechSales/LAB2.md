# LAB2: プログラムの修正と影響分析 🔧

**所要時間**: 10-15分
**難易度**: ★★☆☆☆（初中級）
**使用モード**: Code モード

## 🛠️ ステップ4: RPGプログラムの修正（5分）

### 4.1 元のプログラムからの変更点を確認

**Bobへの質問例**:
```
iph210.rpgを参考に、iph210l2.rpgを以下の要件で修正してください：

1. 画面ファイル名をiph210l2.dspfに変更
2. 郵便番号（TKPOST）と電話番号（TKTELE）の初期化処理を追加
3. 得意先番号（TKBANG）で検索するように変更（現在は仮名検索）

RPG III画面開発の規約（.bob/rules/rules-code/rpg3-screen-development.md）を
参照して、適切な修正を提案してください。
```

**期待される回答のポイント**:
- H仕様書の追加が必要かどうかの確認
- ファイル定義の変更（画面ファイル名、検索ファイル）
- 検索ロジックの変更（TKNAKN → TKBANG、TOKMSL02 → TOKMSP）
- サブルーチン名の確認（特殊文字の使用可否）
- 初期化処理の追加

### 4.2 具体的な修正内容の確認

**Bobへの質問例**:
```
iph210l2.rpgの修正内容を、修正前と修正後のコードで示してください。
特に以下の点を明確にしてください：

1. F仕様書のファイル定義
2. 検索ロジック（SETLL、READ命令）
3. 初期化サブルーチン
```

**期待される回答例**:
```rpg
修正前:
     F* IPH210 * 得意先探索
     FIPH210S CF  E                    WORKSTN
     F                                        RRN01 KSFILE PANEL80
     FTOKMSL02IF  E           K        DISK

修正後:
     H*****************************************************************
     H* IPH210L2 * 得意先探索
     H*****************************************************************
     H            Y
     F* IPH210L2 * 得意先探索
     FIPH210L2CF  E                    WORKSTN
     F                                        RRN01 KSFILE PANEL80
     FTOKMSP  IF  E           K        DISK
```

**重要な変更点**:
1. H仕様書の追加（日付編集コード `Y` 付き）
2. ファイル名を8文字に短縮（IPH210L2）
3. TOKMSL02（仮名検索）→ TOKMSP（番号検索）に変更

### 4.3 修正の実行

**Bobへの依頼例**:
```
提案された修正内容をiph210l2.rpgに適用してください。
```

**期待される回答例**:
```rpg

C                     MOVELTOKTAN    TKNAKN

修正後（検索ロジック）:
C                     MOVEL*BLANK    TKBANG
C                     MOVELTOKTAN    TKBANG
C           TKBANG    SETLLTOKMSP               30
C                     READ TOKMSP                   50

修正前（CLEARサブルーチン）:
C           ¥CLEAR    BEGSR
C                     MOVE *BLANK    TKBANG
C                     MOVE *BLANK    TKNAKN
C                     MOVE *BLANK    TKADR1
C                     MOVE *BLANK    TKADR2
C                     Z-ADD0         TKGEND
C                     Z-ADD0         TKUZAN
C                     ENDSR

修正後（CLEARサブルーチン）:
C           CLEAR     BEGSR
C                     MOVE *BLANK    TKBANG
C                     MOVE *BLANK    TKNAKN
C                     MOVE *BLANK    TKADR1
C                     MOVE *BLANK    TKADR2
C                     MOVE *BLANK    TKPOST
C                     MOVE *BLANK    TKTELE
C                     Z-ADD0         TKGEND
```

**重要な変更点**:
1. サブルーチン名: ¥CLEAR → CLEAR（RPG IIIでは特殊文字不可）

## 🎯 よくある質問

**Q: 画面レイアウトが崩れた場合は？**

A: Bobに「レイアウトを調整してください」と再依頼しましょう。桁位置を具体的に指定して修正を依頼すると、より正確な結果が得られます。コンパイルリストをBobに渡して修正するのもよいです。

**Q: RPG IIIのファイル名が8文字を超えてエラーになった場合は？**

A: RPG IIIでは、F仕様書のファイル名は最大8文字です。`IPH210SL2`（9文字）のような名前は使用できません。`IPH210L2`（8文字）のように短縮してください。

**Q: 元のプログラムは仮名検索だったのに、なぜ番号検索に変更したのですか？**

A: 画面には「得意先探索コード」と表示されており、ユーザーは得意先番号（例：07300）を入力します。元のプログラムは仮名フィールド（TKNAKN）で検索していたため、番号では正しく検索できませんでした。TOKMSP（物理ファイル）を使用することで、得意先番号での検索が可能になります。
